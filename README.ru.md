# codebase-memory-mcp — внутренний форк (сборка из исходников)

Русская инструкция к доработанному варианту. Полное техническое описание
(MCP-инструменты, модель графа, Cypher, языки) — в [README.md](README.md).

**Что это:** MCP-сервер «памяти кодовой базы» для AI-агентов (Claude Code,
Codex CLI, Gemini CLI и др.). Индексирует репозиторий через tree-sitter в
персистентный граф знаний (функции, классы, вызовы, HTTP-маршруты) и отвечает
на структурные запросы за миллисекунды. 14 MCP-инструментов, один статический
бинарник, никаких внешних зависимостей в рантайме.

## Чем этот форк отличается от апстрима

Форк подготовлен для работы в корпоративной среде со строгими политиками
безопасности:

| Было (апстрим) | Стало (этот форк) |
|---|---|
| Фоновая проверка обновлений: `curl` к `api.github.com` при каждом старте MCP-сессии | Удалена полностью — **бинарник не делает ни одного исходящего соединения** |
| Команда `update` скачивала бинарник с GitHub Releases | Отключена: печатает «пересоберите из исходников» и завершается с ошибкой |
| `install.sh` / `install.ps1` скачивали готовую сборку с GitHub | Собирают бинарник из исходников **этого** репозитория, офлайн |
| CI, реестры MCP, пакетные менеджеры (npm/pypi/brew/…), сайт | Удалены — для внутреннего использования не нужны |

Проверить отсутствие сетевой активности можно самостоятельно:
`scripts/security-network.sh` (strace-тест: любое соединение кроме
localhost/DNS — провал) и `scripts/security-strings.sh` (аудит строк бинарника:
никаких URL кроме localhost и лицензионной атрибуции vendored-кода).

## Требования

Сборка стандартного варианта полностью офлайн — все C-зависимости вендорены
в репозитории. Нужны только:

- **Linux / WSL / macOS:** `gcc`/`g++` (или clang) и GNU `make`
  - Debian/Ubuntu: `sudo apt install build-essential`
  - RHEL/Fedora: `sudo dnf install gcc gcc-c++ make`
- **Windows (нативно):** тулчейн MSYS2 —
  `pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-g++ make`
- **Вариант с UI** (`--ui`, 3D-визуализация графа): дополнительно `node` и
  `npm` с доступом к npm registry (публичному или внутреннему зеркалу).
  Стандартному варианту это не нужно.

## Установка

Склонируйте репозиторий из внутреннего git и запустите установщик из корня:

**Linux / WSL / macOS:**
```bash
git clone <внутренний-git-URL>/codebase-memory-mcp.git
cd codebase-memory-mcp
./install.sh
```

**Windows (PowerShell):**
```powershell
git clone <внутренний-git-URL>/codebase-memory-mcp.git
cd codebase-memory-mcp
powershell -ExecutionPolicy ByPass -File install.ps1
```

Установщик: проверяет тулчейн → собирает бинарник (`build/c/codebase-memory-mcp`)
→ копирует его в `~/.local/bin` (Windows: `%LOCALAPPDATA%\Programs\codebase-memory-mcp`)
→ автоматически настраивает все найденные кодинг-агенты (MCP-записи, инструкции,
хуки) → добавляет каталог в PATH (Windows).

**Опции** (одинаковые у install.sh и install.ps1):

| Опция | Действие |
|---|---|
| `--ui` | Собрать вариант со встроенной 3D-визуализацией графа |
| `--standard` | Стандартный вариант без UI (по умолчанию) |
| `--dir=<путь>` | Установить бинарник в другой каталог |
| `--skip-config` | Только собрать и установить бинарник, агентов не трогать |

После установки перезапустите кодинг-агента и скажите ему: **«Index this
project»** — готово.

Windows-альтернатива через WSL: `scripts/setup-windows.ps1 -FromSource`
(копирует исходники в WSL, собирает там и прописывает MCP-запись через
`wsl.exe`).

## Обновление

Самообновления в этом форке нет — бинарник не имеет права ходить в сеть.
Обновление = забрать свежие исходники и пересобрать:

```bash
git pull
./install.sh
```

## Проверка для службы безопасности

Если потребуется показать, что инструмент «чистый»:

```bash
# 1. Сборка воспроизводима из читаемых исходников этого репозитория
./install.sh --dir /tmp/cbm-test --skip-config

# 2. Ноль исходящих соединений (Linux, требует strace)
scripts/security-network.sh /tmp/cbm-test/codebase-memory-mcp

# 3. В бинарнике нет посторонних URL
scripts/security-strings.sh /tmp/cbm-test/codebase-memory-mcp

# 4. Аудит исходников на опасные вызовы (popen/exec) против allowlist
scripts/security-audit.sh
```

Вся обработка локальная: индексы и настройки лежат в
`~/.cache/codebase-memory-mcp` (переопределяется переменной `CBM_CACHE_DIR`),
код проекта никуда не отправляется, телеметрии нет.

## Ручная настройка агента (без команды install)

Добавьте в `~/.claude/.mcp.json` (глобально) или `.mcp.json` проекта:

```json
{
  "mcpServers": {
    "codebase-memory-mcp": {
      "command": "/полный/путь/к/codebase-memory-mcp",
      "args": []
    }
  }
}
```

Перезапустите агента и проверьте `/mcp` — сервер должен отдавать 14
инструментов.

## Основные MCP-инструменты

| Инструмент | Назначение |
|---|---|
| `index_repository` | Проиндексировать репозиторий в граф |
| `search_graph` | Поиск узлов (функции, классы, маршруты) по имени/типу |
| `query_graph` | Cypher-запросы к графу (openCypher, read-only) |
| `trace_path` | Цепочки вызовов между функциями |
| `get_architecture` | Обзор архитектуры проекта |
| `search_code` | Текстовый поиск по коду |
| `get_code_snippet` | Исходник сущности по qualified name |
| `detect_changes` | Что изменилось с последней индексации (git) |
| `index_status` | Покрытие индексом, статистика |

Полный список и модель данных — в [README.md](README.md).

## Удаление

```bash
codebase-memory-mcp uninstall
```

Убирает записи из конфигов агентов, хуки и инструкции. Бинарник и базы
индексов (`~/.cache/codebase-memory-mcp`) удаляются вручную.
